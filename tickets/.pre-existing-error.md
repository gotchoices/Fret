## Why this is pre-existing / not mine

This ticket (`5-sim-metrics-ci-profiles`) touches only the deterministic, in-memory
**simulation harness** (`test/simulation/sim-metrics.ts`, `test/simulation/fret-sim.ts`),
`test/sim-profiles.spec.ts`, and a `.gitignore` line. None of that is imported or
exercised by `proactive-announce.spec.ts`, which spins up real libp2p memory-transport
nodes through `fret-service`. The error is a libp2p stream-lifecycle race
(`StreamStateError: Cannot write to a stream that is closed`) plus the classic Mocha
async-flake symptom (`done() called multiple times`) — both originate inside the
libp2p/yamux stack, not in any code this ticket modified. My own targeted spec
(`test/sim-profiles.spec.ts`) and `npx tsc --noEmit` both pass cleanly.
