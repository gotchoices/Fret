# Pre-existing test failure

## Command
From `packages/fret/`:
```
yarn test
```
(`yarn test` runs mocha with no `--timeout`, so the default 2000ms per-test timeout applies.)

## Failing test
```
Deterministic replay
  two runs with same seed produce identical metrics:
    Error: Timeout of 2000ms exceeded.
    at process.processImmediate (node:internal/timers)
  (test/message-bus.spec.ts)
```

## Why this is pre-existing / not caused by this ticket
- The failure is in `test/message-bus.spec.ts` (the deterministic-simulation harness),
  a subsystem the `seed-new-peers` change never touches. This ticket only edits
  `src/service/fret-service.ts` (sample selection + size-estimator calibration) and
  `test/seed-new-peers.spec.ts`.
- Run in isolation the same test **passes** at ~1166ms:
  `node --import ./register.mjs node_modules/mocha/bin/mocha.js "test/message-bus.spec.ts" --timeout 30000`
  → `20 passing (11s)`.
- It is a load-induced timeout: the test legitimately needs >1s and, under full-suite
  CPU contention, occasionally crosses the 2000ms default. The root issue is the suite's
  per-test timeout budget, not a logic regression.

## Work-around applied
None. No code or test was disabled. The seed-new-peers tests all pass; this flaky
simulation timeout is left untouched for triage.
